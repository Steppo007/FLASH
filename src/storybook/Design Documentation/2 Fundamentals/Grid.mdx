### Grid anatomy

The layout grid structure has three elements:

-   **Columns** divide the page into equal vertical sections, and are used to organize content.

-   **Gutters** are the gaps between the columns, and separate content in a consistent way.

-   **Margins** define the outer edges of the grid, and prevent content from spilling out over the viewable regions.


![image](/images/grid/grid_anatomy.webp)

This grid documentation is for design only. Engineering should refer to the [page layout](https://atlassian.design/components/page-layout/examples "https://atlassian.design/components/page-layout/examples") component.

## Grid types

There are two types of grid: fluid and fixed. Both grid types use breakpoints to determine whether the layout needs to change between viewport sizes.

More guidance for [applying grid](https://atlassian.design/foundations/grid-beta/applying-grid/) in your designs.

### Fluid

The fluid grid stretches across the screen to fill all available space in the main content area. This means the columns and the content placed within them scale and resize to fill the available space.

### Fixed

The fixed grid applies the ideal maximum width to page containers according to the elements being displayed. There is both a fixed-narrow and fixed-wide grid, so content can be displayed using the most appropriate width.


<div style={{display: 'flex', justifyContent: 'space-around', gap: 'var(--space-4)', flexWrap: 'wrap'}}>

  <img src="/images/grid/fixed_grid_medium.webp" style={{ maxHeight: '600px' }} />

**Fixed-narrow** grids are best used for content-focused pages, such as blogs and articles, to limit line length and improve readability. Some internal elements, such as headers, may break out of the grid to span across the page.

## Responsive Breakpoints

Each [breakpoint] is the threshold at which the website's layout changes, to ensure optimal user experience. In responsive design, a breakpoint range determines the most appropriate number of columns for that viewport size, plus recommended widths for margins and gutters.

## Column span and offset

### Column span

Our grid has 12 columns for [aligning content](https://atlassian.design/foundations/grid-beta/applying-grid/#step-4-align-content-to-grid). Content should span across 3 or more columns, up to a maximum of 12. For flexibility, mix and match the number of columns used to unlock a variety of layouts across different breakpoints.

![image](/images/grid/column_span.webp)

### Column offset

Content does not need to span across all 12 columns. It can span a smaller set of columns, resulting in a layout with content centered on a 12 column grid.

![image](/images/grid/column_offset.webp)

## Layout anatomy

Layout regions are the building blocks for any web page. They are composed of elements and components that share similar functions. The layout grid starts below the top navigation, and provides structure to the main content.

![image](/images/grid/layout_anatomy.webp)

### Side navigation and drawer

Use a drawer to display supporting content on a screen, such as help panels or secondary tasks. The layout grid typically fills the full width of the viewport, but when the side navigation or drawer is present, the grid should resize to fit the remaining area of the page.
