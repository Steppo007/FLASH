## Side Navigation

### Overview

A **Side Navigation** component is a compact and vertically positioned UI element along the edge of a webpage or application. It offers quick access to various sections or features, enhancing user navigation efficiency. **Side Navigation** is particularly useful for applications with multiple sections or features that users need to access frequently.

* * * * *

### When to Use It

-   To provide quick access to various sections or features of an application.

-   To organize navigation options in a compact and accessible manner.

-   To enhance the user experience by making navigation intuitive and efficient.

#### **Avoid using side navigations when:**

-   The navigation structure is very simple and does not require dedicated side navigation.

-   The side navigation would clutter the interface and overwhelm users with too many options.

-   The navigation is better suited for a different layout, such as a top navigation bar or footer.

### Variants

-   **Collapsed:** A compact version of the side navigation that displays only icons, saving space and providing a minimalist look.

-   **Expanded:** An expanded version of the side navigation that displays both icons and labels, providing more context and clarity for users.

* * * * *

### Behavior

-   **Default**: The side navigation displays the navigation items with appropriate styling and layout.

-   **Highlighted**: Navigation items appear highlighted to indicate they are currently selected or important.

* * * * *

### Dos and Don'ts

#### **Dos:**

✔ Use side navigation to organize and present navigation options in a compact and accessible manner.\
✔ Ensure side navigation is easily distinguishable by content type and purpose.\
✔ Provide clear and concise labels or icons for each navigation item.

#### **Don'ts:**

✘ Don't overload the side navigation with too many items, which can overwhelm users.\
✘ Don't rely solely on color to indicate interactivity---use additional visual cues like hover effects.\
✘ Don't use side navigation for actions or commands that are better suited for a different layout, such as a top navigation bar or footer.
