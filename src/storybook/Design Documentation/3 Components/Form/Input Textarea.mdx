## Input TextArea

### Overview

An **Input TextArea** Field allows users to enter and edit multi-line text. It is commonly used for comments, descriptions, and other longer text entries.

* * * * *

### When to Use It

-   To collect detailed user input, such as comments or descriptions.

-   When users need to enter multi-line text.

-   For feedback forms, message boxes, and content creation.

#### **Avoid using a Checkbox when:**

-   The input is limited to a single line of text (use a single-line text input instead).

-   The input requires a specific format, such as dates or numbers (use specialized input fields).

### Variants

-   **Standard TextArea**: A basic multi-line text input field.

-   **Resizable TextArea**: Allows users to resize the text area to fit their content.

-   **Fixed Size TextArea**: A text area with a fixed size that cannot be resized.

* * * * *

### Behavior

-   **Hover**: A visible outline color change appears to indicate the field is active and ready for user input.

-   **Focus**: A visibly shaded outline appears to indicate the field is active and ready for input.

-   **Active:** The input field shows a cursor to indicate it is being clicked.

-   **Filled in:** The input field displays the entered text, showing that the user has provided input.

-   **Disabled**: The input field appears grayed out and is non-interactive, preventing any user input.

-   **Disabled + Filled in:** The input field appears grayed out but still displays the previously entered text, indicating it is non-interactive but contains data.

-   **Error State**: The input field changes border color to indicate an error, often accompanied by an error message to guide the user.

* * * * *

### Dos and Don'ts

#### **Dos:**

✔ Use clear and concise placeholder text to guide users.\
✔ Validate user input and provide helpful error messages.\
✔ Ensure the text area is large enough for the expected content.

#### **Don'ts:**

✘ Don't use text areas for short, single-line inputs (use single-line text inputs).\
✘ Don't leave text areas without labels or placeholder text.\
✘ Don't rely solely on color to indicate errors---use text or icons as well.
