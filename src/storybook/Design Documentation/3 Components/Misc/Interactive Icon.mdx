## Interactive Icon

### Overview

An **Interactive Icon** is a graphical element that users can interact with to perform actions or navigate within an interface. These icons are often used for buttons, links, and other interactive elements.

* * * * *

### When to Use It

-   To provide a visual representation of an action or function.

-   To enhance the user interface with intuitive, clickable elements.

-   To save space by using icons instead of text labels.

#### **Avoid using an Interactive Icon when:**

-   The interface requires detailed instructions or explanations (use text instead).

-   The icon is purely decorative and does not provide any interactive functionality.

### Variants

-   **Standard Icon**: A basic interactive icon.

-   **Icon with Tooltip**: An icon that displays a tooltip on hover to provide additional information.

* * * * *

### Behavior

-   **Hover:** The icon changes color to indicate it is interactive and can be clicked.

-   **Focus:** A visible border appears around the icon to indicate it is focused and ready for user interaction.

-   **Active:** The icon shows a tooltip.

-   **Disabled:** The icon appears grayed out and is non-interactive, preventing any user action.

* * * * *

### Dos and Don'ts

#### **Dos:**

✔ Use clear and recognizable icons that convey their function.\
✔ Ensure icons are accessible, with keyboard navigation and screen reader support.\
✔ Provide tooltips or labels for icons that may not be immediately understood.

#### **Don'ts:**

✘ Don't use icons for actions that require detailed explanations (use text labels instead).\
✘ Don't rely solely on color to indicate interactivity---use additional visual cues like hover effects.\
✘ Don't use interactive icons purely for decoration without providing any functionality.
